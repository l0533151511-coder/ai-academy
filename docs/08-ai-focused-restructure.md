# עדכון עדיפויות — מיקוד AI (2026-07-12)

## למה השינוי
המטרה המרכזית של המשתמש: **להפוך למפתח AI מקצועי דרך Claude Code**. תוכנית האב המקורית ([04-master-plan.md](04-master-plan.md)) בנתה מסלול "פול-סטאק" מלא (HTML→CSS→JS→TS→React→Next→Node→SQL→Docker...) לפני שמגיעים ל-AI בכלל. זה נכון לאקדמיה כללית, אבל לא למטרה הספציפית הזו — יותר מדי זמן על נושאים שאינם תורמים ישירות.

**השינוי:** יסודות מוצקים אך מצומצמים → קפיצה מהירה לעולם ה-AI, עם משקל כבד בהרבה על הטראקים 6-10 המורחבים.

## מה נשאר בלי שינוי (כבר בנוי ואומת)
- ✅ מודול 0.1 — יסודות מחשב (5/5 שיעורים)
- ✅ מודול 0.2 — איך האינטרנט עובד (5/5 שיעורים)
שני אלה חיוניים גם למפתח AI (להבין מה זה API/HTTP/שרת זה תנאי סף להבנת מערכות AI) ונשארים כפי שהם.

## מה מצטמצם (מיסודות בלבד, לא נמחק — רק נדחס)
- **מודול 0.3 — טרמינל ו-Linux:** במקום 5 שיעורים מלאים → **2-3 שיעורים ממוקדים**: פקודות ליבה + סימולטור טרמינל, בדיוק כמות ה"שורת פקודה" שנחוצה כדי לעבוד יעיל עם Claude Code (ניווט, קבצים, הרצת סקריפטים). לא עומק Linux מערכתי.
- **מודול 0.4 — Git ו-GitHub:** במקום 5 שיעורים מלאים → **2-3 שיעורים ממוקדים**: commit/branch/PR ברמה שמאפשרת לעבוד בזרימת עבודה עם Claude Code (agentic coding + git workflows). לא עומק rebase/conflict-resolution מתקדם.
- **קפסטון #1** (תיק עבודות גיט) — נשאר אך מצומצם לפרויקט קטן יותר.

## מה מוסר מהמסלול החובה (הופך לנספח אופציונלי, [09-optional-fullstack-appendix.md](09-optional-fullstack-appendix.md))
כל טראק 1-4 המקוריים (Web Fundamentals, Frontend Engineering, Backend Engineering, DevOps וארכיטקטורה — כ-15 מודולים של HTML/CSS/JS/TS/React/Next.js/Node/SQL/Prisma/Docker/System Design) **לא נבנים כברירת מחדל**. הם עוברים לנספח אופציונלי למי שירצה מסלול פול-סטאק מלא בעתיד.

**במקומם — טראק חדש ומצומצם:**

### טראק חדש 1' — יסודות תכנות ל-AI (Programming Essentials for AI Builders)
מטרה: הכי מעט תכנות שמאפשר לבנות ולהבין מערכות AI אמיתיות — לא שליטה בפרונט-אנד/בק-אנד מקצועית.
1. **מודול 1'.1 — JavaScript/TypeScript יסודי** (2-3 שיעורים): משתנים, פונקציות, async/await, fetch — כמה שצריך כדי לקרוא/לכתוב קוד עם Claude Code ולהבין מה הוא עושה.
2. **מודול 1'.2 — Python יסודי** (2-3 שיעורים): רוב עולם ה-AI/ML כתוב בפייתון (LangChain, transformers, scripts) — תחביר בסיסי, קריאת קוד AI קיים.
3. **מודול 1'.3 — REST APIs ומסדי נתונים בקצרה** (2 שיעורים): קריאה/כתיבה ל-API, JSON, ומושג בסיסי על DB (לא בניית backend מלא — רק הבנה מספקת לחיבור מערכות AI לשירותים חיצוניים).
4. **פרויקט מודול:** סקריפט קטן שמפעיל קריאת API אמיתית (למשל ל-Claude API) ומעבד תשובה.

זה מחליף לחלוטין את טראקים 1-4 המקוריים (~15 מודולים) ב-3-4 מודולים ממוקדים.

## מה מתרחב משמעותית (הליבה החדשה של האקדמיה)
הטראקים 5-10 המקוריים נשארים באותם שמות אך **כל אחד מקבל מודולים נוספים** (יפורטו במסמך מעודכן שאבנה כשנגיע אליהם בפועל, כדי לא לתכנן לעומק דברים רחוקים מדי):
- **טראק AI Foundations** — נשאר, אך LLMs (מודול 5.3) מקבל עוד 2-3 שיעורים (fine-tuning בסיסי, מודלים open-source מול API, evaluation).
- **טראק Prompt Engineering + Claude Code** — **הרחבה הגדולה ביותר**: מודול Claude Code מוכפל ל-2 מודולים מלאים (workflows מתקדמים, subagents, hooks, MCP servers מותאמים, אוטומציה של תהליכי פיתוח שלמים, ניפוי שגיאות בעזרת AI).
- **טראק AI Integration Engineering** — MCP, Tool/Function Calling, Embeddings, Vector DBs, RAG — כל אחד מקבל מודול משלו במקום להיות מודול משותף (4→6-7 מודולים).
- **טראק AI Agents** — סוכן בודד, multi-agent, אוטומציה — מורחב עם עוד תרגילי בנייה מעשיים (agent frameworks אמיתיים).
- **טראק Production AI + SaaS** — נשאר כפרויקט הגמר, אך מודגש דיפלוי אמיתי (Vercel/cloud) של מוצרי AI מסחריים.

תוכנן מודול Python נפרד, אך הרצת Python אמיתי בדפדפן דורשת Pyodide (WASM כבד, טעינה איטית). כדי לא לפגוע בחוויית "כל דבר אינטראקטיבי ורץ בפועל", נדחה השילוב המלא של Python לשלב מאוחר יותר (כשנגיע לתכנים שבאמת חייבים אותו, כמו LangChain/transformers). כרגע JS/TS מספיקים לכל מה שנבנה באקדמיה עצמה, כולל קריאות אמיתיות ל-Claude API.
