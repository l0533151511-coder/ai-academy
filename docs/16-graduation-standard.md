# תקן הבוגר — Graduation Standard

מסמך זה מגדיר במדויק מה בוגר של אקדמיית ה-AI **יודע לעשות** — לא אילו שיעורים עבר, אלא אילו
יכולות מוכחות יש לו. הרף שאפתני בכוונה: בוגר צריך להיות מסוגל **לבנות ולתחזק מערכות AI בפרודקשן**
בביטחון, לא רק להסביר מושגים.

הבוחן האמיתי: *"האם הייתי סומך על הבוגר הזה שיקח פיצ'ר AI מרעיון ועד פרודקשן יציב, לבד?"*
היעד של התקן הזה הוא שהתשובה תהיה **כן**.

---

## עקרון-על

בוגר לא "יודע AI" — הוא **חושב כמו מהנדס AI מנוסה**: מתכנן לפני שממש, מודד לפני שמאופטם,
מאבטח לפני שמשיק, ומעדיף את הפתרון הזול והיציב על המרשים. הוא יודע גם *מתי לא* להשתמש ב-AI.

---

## 1. כישורים טכניים (Technical)

בוגר מסוגל, ללא עזרה:
- לקרוא ל-Claude API נכון (system/user, messages, streaming, tool_use) ולטפל בשגיאות/rate-limits.
- לכתוב פרומפט כמפרט חד-משמעי ולקבל פלט **עקבי על מגוון קלטים**, לא רק פעם אחת.
- לאכוף **פלט מובנה** (JSON) עם schema + ולידציה + retry — ולא לסמוך על המודל.
- לבנות **RAG** מקצה-לקצה: chunking, embeddings, retrieval, grounding, ולהעריך רטריבל וגנרציה בנפרד.
- להגדיר **כלים (tool/function calling)** בטוחים ולחבר **MCP** כשצריך שימוש-חוזר חוצה-אפליקציות.
- לבנות **סוכן** עם lלולאת Think-Act-Observe, stop-condition, step-budget ו-human-in-the-loop.
- לבחור בין **prompting → RAG → fine-tuning** לפי ההקשר, ולנמק.

## 2. כישורים הנדסיים (Engineering)

- מפרק משימה גדולה למשימות קטנות, הפיכות וניתנות-לסקירה.
- עובד מאחורי טסטים; כל שינוי מאומת ב-build + typecheck + בדיקה חיה לפני commit.
- כותב קוד שנקרא כמו הקוד שסביבו (עקביות על מקוריות).
- מקבל החלטות trade-off מודעות (עלות/ביצועים/מורכבות) ומתעד את ה"למה".

## 3. יכולת דיבוג (Debugging)

- מגיב ל"זה לא עובד" בשאלות הנכונות: repro, הקשר, השערות לפני תיקון.
- מייצר טסט שמשחזר את הבאג לפני שמתקן.
- מנפה מערכת AI לא-דטרמיניסטית: מבדיל בין באג בקוד, פרומפט חלש, ורטריבל גרוע.
- קורא diff שנוצר ע"י AI בעין ביקורתית ומזהה מה שבור/מומצא.

## 4. יכולת ארכיטקטורה (Architecture)

- מתכנן architecture-first: מגדיר גבולות, זרימת-נתונים ונקודות-כשל לפני שכותב קוד.
- בוחר מודל בידוד נכון (למשל multi-tenancy: tenant_id vs schema vs DB) ומנמק blast-radius.
- מזהה מתי single-agent מספיק ומתי multi-agent מוצדק (ומתי הוא over-engineering).

## 5. עיצוב מוצר AI (AI Product Design)

- ממפה persona ו-jobs-to-be-done; לא בונה "לכולם".
- מתמחר מוצר AI מתוך מודעות ל-margin (עלות טוקן משתנה מול הכנסה קבועה).
- מגדיר MVP, מדד-הפעלה (activation), ו-time-to-value.

## 6. שליטה ב-Claude Code (Claude Code Mastery)

- מנווט codebase גדול ולא-מוכר: מבקש מפת-ארכיטקטורה, נקודות-כניסה, זרימה — לפני שינוי.
- מנהל הקשר (CLAUDE.md, scoping, סשן נקי) ומונע context rot.
- מוביל את Claude Code בפרומפט חזק (מפרט, לא משאלה), סוקר את הפלט, ומשלח בבטחה.
- משתמש בו לעבודה יומית אמיתית: תכנון, בנייה, דיבוג, refactor, טסטים, סקירה, תחזוקה.

## 7. מוכנות לפרודקשן (Production Readiness)

- מבדיל בין "עובד" ל"מוכן ללקוחות משלמים" (טכני/אבטחה/עסקי/תיעוד).
- מיישם graceful degradation — המערכת עובדת (או נכשלת בחן) גם כשה-AI לא זמין.
- מגן מפני prompt injection ומיישם guardrails + red-teaming כשגרה.
- מפתח באחריות: שקיפות, פיקוח-אנושי לפעולות משמעותיות, מודעות להטיה ולפרטיות.

## 8. פריסה (Deployment)

- מפעיל pipeline: build → test → verify → deploy → verify-prod → rollback.
- לעולם לא מפרס פלט-AI לא-מאומת; משתמש ב-staging, feature flags ו-canary.
- יודע לגלגל אחורה פרומפט/מודל גרוע **בלי** redeploy.

## 9. בדיקות (Testing)

- בונה **evals** (golden set) למערכת לא-דטרמיניסטית במקום assert-equals.
- מריץ regression על כל שינוי פרומפט/מודל ומגדיר gate ב-CI.
- מרחיב את ה-golden set מכשלי-פרודקשן אמיתיים.

## 10. ניטור (Monitoring)

- מתעד כל קריאת API (עלות, טוקנים, latency, הצלחת-כלים) בלוגינג מובנה.
- מגדיר מדדים ייחודיים ל-AI ומתריע על קפיצות עלות/latency/שגיאה.
- מזהה אנומליה מתוך נתונים, לא מתחושת-בטן.

## 11. תקשורת (Communication)

- כותב מפרט, runbook תקרית, ודוח-אבטחה קצרים וברורים.
- מסביר trade-off הנדסי למי-שאינו-מהנדס (עלות מול איכות, סיכון מול מהירות).
- מתעד החלטות כך שמהנדס אחר יבין את ה"למה" בעוד חצי שנה.

## 12. תחזוקה (Maintenance)

- מטפל ב-drift: מזהה שאיכות/עלות השתנו לאורך זמן ומרענן (נתונים, פרומפט, מודל).
- עובד עם קוד legacy ועם קוד-AY לא-מושלם: משפר בהדרגה, מאחורי טסטים.
- מנהל חוב-טכני מודע: מתקן מה שקריטי, מתעד את השאר כ"חוב ידוע" עם נימוק.

## הוכחת היכולת: הפורטפוליו

בוגר לא מציג תעודה — הוא מציג את **AtlasDesk**, מוצר AI שבנה שלב-אחר-שלב לאורך האקדמיה, עם
ארכיטקטורת production אמיתית (זיכרון, כלים, פלט-מובנה, RAG, סוכן, multi-agent, אוטומציה, ניטור,
אבטחה, safety, בקרת-גישה). זו ההוכחה החיה שהוא יכול לבנות מערכת AI אמיתית — לא דוגמת-צעצוע.

**השורה התחתונה:** בוגר יכול לקחת רעיון למוצר AI, לתכנן אותו, לבנותו עם Claude Code, לאבטח,
לבדוק, לפרוס, לנטר ולתחזק אותו בפרודקשן — בביטחון, ולבד.
