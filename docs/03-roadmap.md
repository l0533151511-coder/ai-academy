# תוכנית עבודה — סדר בנייה

**חוק ברזל:** לא בונים הכל בבת אחת. כל מודול נבנה עד גימור מלא (עיצוב+תוכן+תרגילים+בדיקות) לפני שעוברים למודול הבא.

## שלב א' — שלד המערכת (Platform Skeleton)
1. הקמת פרויקט Next.js+TS+Tailwind, חיבור Supabase (auth+db), RTL בסיס
2. בניית `LessonShell` וכל רכיבי הליבה (ריקים/placeholder אך פונקציונליים)
3. Dashboard בסיסי + מפת טראקים + ניווט
4. מנגנון התקדמות/XP/הישגים בסיסי

## שלב ב' — מודול מופת (Reference Module)
בניית **מודול 0.1 — יסודות מחשב** במלואו כ"מודול לדוגמה" שמוכיח את כל הרכיבים: מצגת, אנימציה, דיאגרמה אינטראקטיבית, Playground, תרגיל עם רמזים מדורגים, בוחן, פרויקט, מונחון, שו"ת, חבילת וידאו מלאה (סקריפט+סטוריבורד).
→ **עצירה לאישור המשתמש** לפני המשך לשאר המודולים, כדי לוודא שרמת האיכות תואמת ציפייה לפני שמשקיעים בעשרות מודולים נוספים.

## שלב ג' — המשך טראק 0 ו-1
השלמת שאר מודולי יסודות + ווב פונדמנטלס לפי סדר הקוריקולום.

## שלב ד' ואילך
המשך טראק אחרי טראק לפי סדר [01-curriculum.md](01-curriculum.md), עם AI Mentor אינטגרציה אמיתית (Claude API) נכנסת בשלב מוקדם (סביב שלב ב') כי היא חוצה את כל השיעורים.
